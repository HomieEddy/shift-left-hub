# Shift-Left Knowledge Hub - Database Design Document (DDD)

> **Updated:** 2026-06-23 — v2.2 milestone.
> The schema here reflects the actual baseline (`V1__baseline.sql` + the V4 hygiene
> migration). Earlier revisions of this document mixed legacy column names
> (`requester_id`, `password_hash`, `BIGSERIAL article id`) that no longer match
> the schema — those are corrected below.

## Database Strategy: Postgres-Native Efficiency

The schema relies entirely on PostgreSQL 16 with pgvector. By utilizing native data types (UUIDs, JSONB, TSVECTOR, and vector embeddings), the architecture eliminates the need for separate search engines or complex NoSQL sidecars.

All primary keys are `UUID` (not `BIGSERIAL` — see ticket/article/category/tag). The single exception is the `ticket_number_sequence` table, which uses `BIGINT GENERATED BY DEFAULT AS IDENTITY` to drive the human-readable `TKT-NNNNN` ticket number generator.

## 2. Enumerations (stored as VARCHAR, validated in JPA)

There are no native Postgres ENUMs — all enum-style fields are stored as `VARCHAR(255)` with the allowed values enforced in the JPA entity:

- `user_role`: `'ROLE_USER'`, `'ROLE_AGENT'`, `'ROLE_ADMIN'`
- `ticket.status`: `'NEW'`, `'IN_PROGRESS'`, `'RESOLVED'`, `'CANCELLED'`
- `ticket.category`: `'NETWORK'`, `'HARDWARE'`, `'SOFTWARE'`, `'ACCESS'`, `'PERIPHERALS'`
- `ticket.urgency`: `'LOW'`, `'MEDIUM'`, `'HIGH'`
- `article.status`: `'DRAFT'`, `'PUBLISHED'`, `'ARCHIVED'`
- `document.status`: `'UPLOADED'`, `'PARSING'`, `'CHUNKING'`, `'EMBEDDING'`, `'READY'`, `'FAILED'`
- `workspace_invitation.status`: `'PENDING'`, `'ACCEPTED'`, `'REVOKED'`
- `workspace_member.role`: `'ADMIN'`, `'MEMBER'`, `'READ_ONLY'`

## 3. Core Tables

### `workspace`

Multi-tenant isolation root. All domain entities reference this.

- `id`: UUID (PRIMARY KEY)
- `name`: VARCHAR(255) (NOT NULL)
- `slug`: VARCHAR(100) (UNIQUE, NOT NULL)
- `description`: TEXT
- `logo_url`: VARCHAR(512)
- `icon`: VARCHAR(64) (Lucide icon name)
- `created_by`: UUID (FK → app_user, NOT NULL)
- `created_at`, `updated_at`: TIMESTAMP
- `deleted_at`: TIMESTAMP (soft-delete support)

### `workspace_member`

Maps users to workspaces with scoped roles.

- `workspace_id`: UUID (FK → workspace, composite PK part 1)
- `user_id`: UUID (FK → app_user, composite PK part 2)
- `role`: VARCHAR(20) (`'ADMIN'`, `'MEMBER'`, `'READ_ONLY'`)
- *Composite primary key:* `(workspace_id, user_id)`

### `workspace_invitation`

In-app member invitations.

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `invited_email`: VARCHAR(255)
- `role`: VARCHAR(20)
- `invited_by`: UUID (FK → app_user)
- `status`: VARCHAR(20)
- `token`: VARCHAR(64) (UNIQUE, for accept/revoke link)

### `app_user`

Authentication and user management.

- `id`: UUID (PRIMARY KEY)
- `email`: VARCHAR(255) (UNIQUE, NOT NULL)
- `password`: VARCHAR(255) (NOT NULL) — BCrypt hash, not the literal password
- `display_name`: VARCHAR(255) (NOT NULL)
- `role`: VARCHAR(255) (global role — see Enumerations)
- `default_workspace_id`: UUID (FK → workspace)
- `enabled`: BOOLEAN
- `created_at`, `updated_at`: TIMESTAMP

### `article` (The Knowledge Base)

- `id`: UUID (PRIMARY KEY) — **not** `BIGSERIAL`
- `workspace_id`: UUID (FK → workspace, NOT NULL)
- `title_en`: VARCHAR(255) (NOT NULL)
- `title_fr`: VARCHAR(255)
- `content_en`: TEXT (NOT NULL)
- `content_fr`: TEXT
- `slug`: VARCHAR(255) (UNIQUE)
- `excerpt`, `excerpt_fr`: TEXT
- `featured_image`: VARCHAR(255)
- `status`: VARCHAR(255) (article_status)
- `author_id`: UUID (FK → app_user, NOT NULL)
- `last_editor_id`: UUID (FK → app_user)
- `source_ticket_id`: UUID (UNIQUE, FK → ticket, for KCS)
- `category_id`: UUID (FK → category)
- `tsv_en`, `tsv_fr`: TSVECTOR (persistent, generated)
- `view_count`: INTEGER (default 0)
- `published_at`, `created_at`, `updated_at`: TIMESTAMP

### `tag`

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `name_en`: VARCHAR(255) (NOT NULL)
- `name_fr`: VARCHAR(255) (NOT NULL)
- `color`: VARCHAR(255) (NOT NULL) — hex color, no length cap at the DB level
- `created_at`: TIMESTAMP
- **Unique constraint (V4):** `(workspace_id, name_en)`

### `article_tag`

Many-to-many join.

- `article_id`: UUID (FK → article)
- `tag_id`: UUID (FK → tag)
- *Composite primary key:* `(article_id, tag_id)`

### `category`

Domain taxonomy for articles and documents.

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `name_en`, `name_fr`: VARCHAR(255) (NOT NULL)
- `parent_id`: UUID (self-referencing FK for subcategories)
- `sort_order`: INTEGER
- `created_at`, `updated_at`: TIMESTAMP
- **Unique constraint (V4):** `(workspace_id, parent_id, name_en)` with `NULLS NOT DISTINCT` so root categories (parent_id IS NULL) are also unique by name

### `document`

Uploaded documents for the ingestion pipeline.

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `filename`: VARCHAR(255)
- `mime_type`: VARCHAR(100)
- `file_size`: BIGINT
- `content_hash`: VARCHAR(64) (SHA-256)
- `status`: document_status
- `error_message`: TEXT
- `uploaded_by`: UUID (FK → app_user)
- `category_id`: UUID (FK → category)
- `created_at`, `updated_at`: TIMESTAMP

### `document_chunk`

Vectorized chunks from document ingestion.

- `id`: UUID (PRIMARY KEY)
- `document_id`: UUID (FK → document, ON DELETE CASCADE)
- `content`: TEXT (NOT NULL)
- `chunk_index`: INTEGER (NOT NULL)
- `embedding`: `vector(${embeddingDimensions})` (pgvector) — dimension templated at migrate time, default 768 (Ollama `nomic-embed-text`)
- `tsv_content`: TSVECTOR (for FTS on document chunks)
- `created_at`: TIMESTAMP

### `ticket` (Incident Management)

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `ticket_number`: VARCHAR(9) (UNIQUE, `TKT-NNNNN` format)
- `user_id`: UUID (FK → app_user, NOT NULL) — requester
- `assigned_to_id`: UUID (FK → app_user)
- `category`: VARCHAR(255) (ticket.category)
- `urgency`: VARCHAR(255) (ticket.urgency)
- `status`: VARCHAR(255) (ticket.status)
- `issue`: TEXT (NOT NULL) — the user's problem statement
- `shift_left_context`: JSONB (AI chat transcript)
- `resolution_notes`: TEXT
- `resolved_by_id`: UUID (FK → app_user)
- `is_knowledge_gap`: BOOLEAN (default false)
- `resolved_at`, `cancelled_at`, `created_at`, `updated_at`: TIMESTAMP
- `cancel_reason`: TEXT

### `work_note`

Agent notes on ticket resolution.

- `id`: UUID (PRIMARY KEY)
- `ticket_id`: UUID (FK → ticket)
- `workspace_id`: UUID (FK → workspace)
- `author_id`: UUID (FK → app_user)
- `content`: TEXT
- `created_at`: TIMESTAMP

### `ai_chat_session`

AI conversation sessions.

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace)
- `user_id`: UUID (FK → app_user)
- `title`: VARCHAR(255)
- `created_at`, `updated_at`: TIMESTAMP

### `ai_chat_message`

Individual messages in AI chat sessions.

- `id`: UUID (PRIMARY KEY)
- `session_id`: UUID (FK → ai_chat_session)
- `role`: VARCHAR(10) (`'user'`, `'assistant'`)
- `content`: TEXT
- `sources`: JSONB (cited sources metadata)
- `created_at`: TIMESTAMP

### `ai_config` (global)

The single global AI configuration (only one row, enforced by service).

- `id`: UUID (PRIMARY KEY)
- `llm_provider`: VARCHAR(255)
- `ollama_endpoint_url`: VARCHAR(255)
- `openai_api_key`: VARCHAR(255) (encrypted at rest)
- `chat_model_name`: VARCHAR(255)
- `embedding_model_name`: VARCHAR(255)
- `similarity_threshold`: DOUBLE PRECISION (bootstrap default 0.35, user-set min 0.65)
- `embedding_dimension`: INTEGER

### `workspace_llm_config` (per-workspace)

Per-workspace LLM endpoint configuration. Sibling of `ai_config`; an active workspace uses one or the other depending on whether the workspace has overridden the global config.

- `id`: UUID (PRIMARY KEY)
- `workspace_id`: UUID (FK → workspace, UNIQUE)
- `llm_provider`: VARCHAR(32) (default `'OLLAMA'`)
- `endpoint_url`: VARCHAR(512)
- `api_key`: VARCHAR(512) (encrypted at rest)
- `model_name`: VARCHAR(128) (default `'llama3.2'`)
- `embedding_model_name`: VARCHAR(128) (default `'nomic-embed-text'`)
- `similarity_threshold`: DOUBLE PRECISION (default 0.65)
- `embedding_dimension`: INTEGER (default 768)
- `system_prompt`: TEXT (customizable with template variables)
- `created_at`, `updated_at`: TIMESTAMP

### `vector_store` (Spring AI managed)

Spring AI's vector_store table is part of the baseline (created at migrate time, not at runtime) so the dimension matches the configured embedding model.

- `id`: UUID (PRIMARY KEY)
- `content`: TEXT
- `metadata`: JSONB
- `embedding`: `vector(${embeddingDimensions})`

## 4. Indexing Strategy

- **FTS:** GIN index on `article.tsv_en`, `article.tsv_fr`, `document_chunk.tsv_content`
- **Vector:** IVFFlat index on `document_chunk.embedding` and `vector_store.embedding`
- **Workspace Isolation:** Composite B-Tree on `(workspace_id, ...)` for all domain tables
- **Lookup:** B-Tree on `workspace.slug`, `article.slug`, `ticket.ticket_number`, `app_user.email`
- **Filtering:** B-Tree on `ticket(status)`, `article(status)`, `document(status)`
- **v2.2 additions (V4):** `idx_ticket_user_id`, `idx_ticket_assigned_to_id`, `idx_ticket_resolved_by_id`, `idx_article_last_editor_id`

## 5. Soft Delete

Only `workspace.deleted_at` is soft-deleted. All other entities use hard delete (or are kept indefinitely — e.g. KCS drafts reference resolved tickets, so tickets should not be deleted while drafts exist). The `findAllNonDeleted()` helper in `WorkspaceService` filters out `deleted_at IS NOT NULL` rows at the service layer.
