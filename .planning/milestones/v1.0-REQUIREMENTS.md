## v1 Requirements — Complete

### Authentication

- [x] **AUTH-01**: User can sign up with email and password — ✓ Validated
- [x] **AUTH-02**: User can log in with credentials and receive a JWT session — ✓ Validated
- [x] **AUTH-03**: User session persists across browser refresh via refresh token rotation — ✓ Validated
- [x] **AUTH-04**: User is assigned a role (ROLE_USER or ROLE_ADMIN) — ✓ Validated

### Knowledge Base

- [x] **KB-01**: Admin can create, edit, and archive knowledge base articles in markdown — ✓ Validated
- [x] **KB-02**: User can browse published articles with title and tag listing — ✓ Validated
- [x] **KB-03**: User can search articles using full-text search with result snippets — ✓ Validated
- [x] **KB-04**: Articles support tags for categorization (Tag entity with M2M) — ✓ Validated
- [x] **KB-05**: Articles are bilingual — separate content per language (EN/FR columns) — ✓ Validated

### AI Self-Service Portal

- [x] **AI-01**: User can describe an IT issue in a conversational chat interface — ✓ Validated
- [x] **AI-02**: System searches knowledge base using hybrid search (FTS + vector) and returns guides — ✓ Validated
- [x] **AI-03**: AI responses stream in real-time via SSE with a typing indicator — ✓ Validated
- [x] **AI-04**: System asks "Did this solve your problem?" after presenting a guide — ✓ Validated
- [x] **AI-05**: If no relevant article found, system returns helpful fallback and offers escalation — ✓ Validated
- [x] **AI-06**: System enforces similarity threshold (>0.65) to prevent hallucinated responses — ✓ Validated

### Escalation & Ticketing

- [x] **TKT-01**: User can escalate an unresolved issue to a human agent — ✓ Validated
- [x] **TKT-02**: Escalation creates a ticket pre-filled with issue and AI chat transcript — ✓ Validated
- [x] **TKT-03**: User selects category and urgency — ✓ Validated
- [x] **TKT-04**: Ticket status tracks through NEW → IN_PROGRESS → RESOLVED — ✓ Validated

### Agent Dashboard

- [x] **AGT-01**: IT agent can view a prioritized list of all tickets — ✓ Validated
- [x] **AGT-02**: Agent can filter tickets by status, category, and urgency — ✓ Validated
- [x] **AGT-03**: Agent can view full ticket detail including shift-left deflection context — ✓ Validated
- [x] **AGT-04**: Agent can add resolution notes and mark ticket as resolved — ✓ Validated
- [x] **AGT-05**: Agent can flag a resolved ticket as a "Knowledge Gap" for KCS drafting — ✓ Validated

### KCS Auto-Drafting

- [x] **KCS-01**: When agent resolves a ticket flagged as Knowledge Gap, system auto-drafts a KB article — ✓ Validated
- [x] **KCS-02**: AI synthesizes article content from the ticket timeline and resolution notes — ✓ Validated
- [x] **KCS-03**: Drafted articles saved with status DRAFT and linked to source ticket — ✓ Validated
- [x] **KCS-04**: System checks for duplicate articles before creating a new draft — ✓ Validated

### Admin Console

- [x] **ADM-01**: Admin can view a queue of AI-drafted articles pending review — ✓ Validated
- [x] **ADM-02**: Admin can edit, approve (→ PUBLISHED), or reject (→ ARCHIVED) draft articles — ✓ Validated
- [x] **ADM-03**: Admin can manage users (view, assign roles) — ✓ Validated
- [x] **ADM-04**: Admin can manage article tags — ✓ Validated

### Infrastructure

- [x] **INF-01**: Local dev environment via Docker Compose (PostgreSQL 16 + pgvector) — ✓ Validated
- [x] **INF-02**: Backend runs on Spring Boot 3.x with Java 21 — ✓ Validated
- [x] **INF-03**: Frontend serves as a standalone Angular SPA — ✓ Validated
- [x] **INF-04**: AI module works with local LLM fallback (Ollama) for API-key-free demo — ✓ Validated

## Out of Scope (Carried Forward)

| Feature | Reason |
|---------|--------|
| Email notifications | Not core to KCS loop, defers to v2 |
| SLA management | Enterprise feature, no portfolio demo value |
| Real-time agent chat | Destroys async workflow benefits, requires staffing |
| Mobile native app | Responsive SPA is sufficient |
| Elasticsearch | Overkill — PostgreSQL FTS + pgvector handles all search needs |
| Kafka/RabbitMQ | Unnecessary infrastructure for modular monolith |
| SSO/OAuth | Setup complexity not justified for portfolio demo |
| Multi-tenancy | Single-tenant portfolio app |
