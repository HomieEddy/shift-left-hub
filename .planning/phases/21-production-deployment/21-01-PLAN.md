    **Step 1 — Create `frontend/public/env.js`**

    Create the file with this exact content:
    ```javascript
    // Runtime environment configuration for Shift-Left Knowledge Hub.
    // Set `window.__env.apiBaseUrl` to the Railway backend URL in production.
    // In development, leave empty — the Angular dev proxy handles routing.
    (window.__env || (window.__env = {})).apiBaseUrl = '';
    ```

    This file:
    - Lives in `public/` so Vercel serves it as a static asset (no build hash)
    - Uses a default empty string (dev proxy mode) — the user replaces `''` with the Railway URL after creating the Railway project (per D-06: domain known after project creation)
    - Uses the self-executing pattern to safely extend window.__env without overwriting existing properties
    - After the user sets the Railway URL, all `/api/` requests will be prefixed

    **Step 2 — Create `frontend/src/app/core/http/base-url.interceptor.ts`**

    ```typescript
    import { HttpInterceptorFn } from '@angular/common/http';

    /**
     * Prepend the production backend API base URL to all /api/ requests.
     *
     * In development (window.__env.apiBaseUrl is empty), requests use relative
     * paths and the Angular proxy.conf.json handles routing.
     *
     * In production, window.__env.apiBaseUrl is set to the Railway backend URL
     * via public/env.js (per D-09).
     */
    export const baseUrlInterceptor: HttpInterceptorFn = (req, next) => {
      const env = (window as unknown as { __env?: { apiBaseUrl?: string } }).__env;
      const baseUrl = env?.apiBaseUrl ?? '';

      if (baseUrl && req.url.startsWith('/api/')) {
        const apiReq = req.clone({ url: `${baseUrl.replace(/\/+$/, '')}${req.url}` });
        return next(apiReq);
      }

      return next(req);
    };
    ```

    Key design decisions:
    - Only prepends when `baseUrl` is non-empty (dev mode still uses proxy)
    - Strips trailing slash from baseUrl to avoid double-slash issues
    - Type-safe access to window.__env via type assertion (not `any`)
    - Minimal runtime cost — single string check per request

    **Step 3 — Register the interceptor in `frontend/src/app/app.config.ts`**

    Read the current `app.config.ts`. It currently has:
    ```typescript
    provideHttpClient(withInterceptors([errorInterceptor])),
    ```

    Change it to:
    ```typescript
    provideHttpClient(withInterceptors([baseUrlInterceptor, errorInterceptor])),
    ```

    **Order matters:** `baseUrlInterceptor` runs FIRST (prepends URL), then `errorInterceptor` handles errors. This ensures errors are reported against the correct full URL.

    Add the import at the top:
    ```typescript
    import { baseUrlInterceptor } from './core/http/base-url.interceptor';
    ```

    **Step 4 — Add env.js script tag to `frontend/src/index.html`**

    Read `frontend/src/index.html`. Add the env.js script BEFORE `</head>` closing tag:
    ```html
    <script src="/env.js"></script>
    ```

    Place it after the `<base href="/" />` line and before the `<meta name="viewport">` line, so it loads early but after the base href is set:
    ```html
    <base href="/" />
    <script src="/env.js"></script>
    ```
