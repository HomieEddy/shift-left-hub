    ```java
    @PostMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter chat(@Valid @RequestBody ChatRequest request, Authentication auth) {
        SseEmitter emitter = new SseEmitter(30_000L); // 30s timeout per D-16, D-21
        
        // Run the AI chat in a separate thread so the emitter can return immediately
        Executors.newSingleThreadExecutor().submit(() -> {
            try {
                aiChatService.processChat(request, emitter, auth.getName());
            } catch (Exception e) {
                try {
                    emitter.send(SseEmitter.event()
                        .name("error")
                        .data(new StreamEvent("error", "An error occurred: " + e.getMessage(), null)));
                    emitter.complete();
                } catch (IOException ex) {
                    emitter.completeWithError(ex);
                }
            }
        });
        
        // Handle client disconnect
        emitter.onCompletion(() -> { /* cleanup if needed */ });
        emitter.onTimeout(() -> {
            try {
                emitter.send(SseEmitter.event()
                    .name("error")
                    .data(new StreamEvent("error", "Request timed out after 30 seconds", null)));
            } catch (IOException e) { /* ignore */ }
            emitter.complete();
        });
        
        return emitter;
    }
    ```
    
    The SSE event format uses `SseEmitter.event().name("message")` for standard SSE. Each event has:
    - `event: message` followed by `data: {json}` (for "token", "done", "error", "fallback", "sources" types)
    - The StreamEvent record serializes to JSON via Jackson automatically

  <behavior>
    - Hybrid search: Given a query, return merged results from FTS (rank) + vector similarity (score) using RRF with equal weights. Results below similarityThreshold (0.65) excluded.
    - Context construction: Given hybrid search results, construct a context string with article titles and content excerpts. Include last 10 messages from chat history in the system prompt.
    - LLM streaming: Given context + user question + history, stream response tokens via SseEmitter.
    - Fallback: When no results above threshold, return StreamEvent with type "fallback" containing the fallback message and escalation flag.
    - "Did this solve?": After completed response, the done event includes the full response text. The frontend handles the yes/no buttons (no server state needed).

    **AiChatService.java**:
    `@Service @RequiredArgsConstructor @Transactional(readOnly = true)`

    Fields injected:
    - `ArticleRepository articleRepository` — for FTS search
    - `VectorStore vectorStore` — Spring AI PGVectorStore for similarity search (auto-configured)

    **HybridSearchResult** (inner record):
    ```java
    record HybridSearchResult(UUID articleId, String titleEn, String titleFr, String slug, String excerpt, double score) {}
    ```

    **Methods:**

    `processChat(ChatRequest request, SseEmitter emitter, String userId)`:
    This is the main orchestration method. Steps:
    1. Extract user message from request, last 10 messages from history
    2. **Hybrid Search**: Call `hybridSearch(request.message())` to get ranked results
    3. **Threshold check**: If top result score < similarityThreshold (from AiConfig) → send fallback event and return
    4. **Context construction**: Format top 3-5 results into a context string (title + excerpt per result)
    5. **Build system prompt**: Combine context, conversation history, and user question into prompt
    6. **Stream response**: Use ChatModel's streaming API (`.stream()`), send each token as `StreamEvent("token", text, null)` via emitter
    7. **Send done event**: After streaming completes, send `StreamEvent("done", fullResponse, sourceRefs)` with article sources
    8. **Handle errors**: Catch exceptions, send error event, complete emitter

    `hybridSearch(String query)` → List&lt;HybridSearchResult&gt;:
    1. **FTS search**: Call `articleRepository.searchByText(query, PageRequest.of(0, 10))` to get FTS results. The results include rank (computed from ts_rank in the SQL).

    2. **Vector search**: Use `vectorStore.similaritySearch(SearchRequest.query(query).withTopK(10))` to get vector similarity results. Each result has a score (cosine distance).
       - In Spring AI 2.0, the VectorStore API is: `vectorStore.similaritySearch(SearchRequest.query(query).withSimilarityThreshold(0.0).withTopK(10))`
       - The results return `List&lt;Document&gt;` where each Document has metadata (articleId, title, slug) and a score

    3. **RRF merge**: Implement equal-weight RRF (k=60 per literature):
       - Create a map from articleId → rank contributions
       - For FTS results (ranked list), assign RRF score contribution: position `i` → `1.0 / (k + i)`
       - For vector results (ranked list), assign RRF score contribution: position `j` → `1.0 / (k + j)`
       - Sum contributions per article
       - Sort by total RRF score descending
       - Return hybrid search results with the computed score

    4. Apply similarity threshold filter: exclude results where RRF score < aiConfigService.getConfigEntity().getSimilarityThreshold()

    `buildContext(List&lt;HybridSearchResult&gt; results)`:
    Format the top results as:
    ```
    Relevant knowledge base articles:
    
    --- Article: {title} ---
    {excerpt or first 500 chars of content}
    [score: {score}]
    ```

    `buildPrompt(String userMessage, String context, List&lt;ChatMessage&gt; history)`:
    System prompt structure (hardcoded for now per D-17):

    ```
    Provide step-by-step resolution guides based on the articles. If the articles don't answer the question,
    say you couldn't find relevant information and offer to escalate.
    
    {context}
    
    Conversation history:
    {formatted history}
    
    User: {message}
    Assistant:
    ```

    **Streaming Implementation**:
    Use Spring AI 2.0's streaming API:
    ```java
    Prompt prompt = new Prompt(new UserMessage(fullPrompt));
    chatModel.stream(prompt).subscribe(chunk -> {
        String text = chunk.getResult().getOutput().getText();
        if (text != null && !text.isEmpty()) {
            StreamEvent event = new StreamEvent("token", text, null);
            emitter.send(SseEmitter.event().name("message").data(event));
        }
    }, error -> {
        // error handling
    }, () -> {
        // complete — send done event with sources
        emitter.send(SseEmitter.event().name("message")
            .data(new StreamEvent("done", fullResponse, sourceRefs)));
        emitter.complete();
    });
    ```
    NOTE: The Spring AI 2.0 streaming API in the ChatModel might use `Flux&lt;ChatResponse&gt;` patterns. The executor should check the actual API signature from the Spring AI 2.0.0-M8 library. Alternatives: use `chatModel.call(prompt)` with `StreamCallback` or the `stream()` method returning `Flux&lt;ChatResponse&gt;`. Use what's available in M8.
