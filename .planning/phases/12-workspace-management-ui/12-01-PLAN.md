  <action>
    Create `backend/src/main/resources/db/migration/V6__add_workspace_invitation_and_icon.sql` with:

    1. CREATE TABLE workspace_invitation:
       - id UUID PRIMARY KEY (NOT NULL)
       - workspace_id UUID NOT NULL (FK → workspace.id)
       - invited_user_id UUID NOT NULL (FK → app_user.id)
       - invited_by UUID NOT NULL (FK → app_user.id)
       - role VARCHAR(32) NOT NULL DEFAULT 'MEMBER'
       - status VARCHAR(16) NOT NULL DEFAULT 'PENDING' — values: PENDING, ACCEPTED, REJECTED
       - created_at TIMESTAMP WITHOUT TIME ZONE NOT NULL

    2. Foreign keys:
       - fk_wsinvitation_workspace → workspace(id)
       - fk_wsinvitation_invited_user → app_user(id)
       - fk_wsinvitation_invited_by → app_user(id)

    3. Indexes:
       - idx_wsinvitation_workspace ON workspace_invitation(workspace_id)
       - idx_wsinvitation_invited_user ON workspace_invitation(invited_user_id)
       - idx_wsinvitation_status ON workspace_invitation(status)

    4. ALTER TABLE workspace:
       - ADD COLUMN deleted_at TIMESTAMP WITHOUT TIME ZONE (nullable, no default — NULL = not deleted)
       - ADD COLUMN icon VARCHAR(64) (nullable — stores Lucide icon name like "building2", "globe", "rocket")

    Do NOT add NOT NULL constraints to the new workspace columns. Keep them fully nullable for backward compatibility with existing rows.

    Use IF NOT EXISTS patterns where possible for idempotency. Reference V5 migration structure for style consistency.
  </action>

  </verify>
