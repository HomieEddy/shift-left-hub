    frontend/src/app/core/auth/workspace-role.service.ts
  </files>
  <action>
    Create `workspace-role.service.ts` under `src/app/core/auth/`:

    - `@Injectable({ providedIn: 'root' })`
    - Inject `HttpClient`
    - Private field: `roleSignal = signal<string>('NONE')`
    - Public readonly: `role = this.roleSignal.asReadonly()`
    - Public computed signals for convenience:
      - `isAdmin = computed(() => this.role() === 'ADMIN')`
      - `isMember = computed(() => this.role() === 'MEMBER' || this.role() === 'ADMIN')` (members and admins can create content per D-22)
      - `isReadOnly = computed(() => this.role() === 'READ_ONLY')`
      - `isAuthenticated = computed(() => this.role() !== 'NONE')`

    - Method `fetchRole(): Observable<WorkspaceRoleResponse>`:
      - HTTP GET `/api/workspaces/current/role` with `{ withCredentials: true }`
      - Pipe tap to update `this.roleSignal.set(response.role)`
      - Return the observable

    - Method `refreshRole(): void`:
      - Calls fetchRole().subscribe() — fire-and-forget for workspace switches

    - Method `clearRole(): void`:
      - Sets roleSignal to 'NONE' — for logout

    Follow the `AuthService` pattern for signal-based state management.
  </action>
