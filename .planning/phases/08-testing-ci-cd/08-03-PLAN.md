  <action>
    Create `AuthFlowIntegrationTest.java` extending `AbstractIntegrationTest`:

    Full lifecycle scenario:
    1. Register a new user → verify 200 + access + refresh tokens returned
    2. Login with credentials → verify 200 + tokens
    3. Access a protected endpoint (e.g., GET /api/tickets) with the access token → verify 200

    Login endpoint: `POST /api/auth/login` with body `{"email":"...","password":"..."}`

    For cookie-based auth, extract `Set-Cookie` header to get the refresh cookie, then pass it in subsequent requests.
    Use `TestRestTemplate` with `HttpComponentsClientHttpRequestFactory` for cookie support.

    Data cleanup: Use `@Sql` or `@DirtiesContext` as needed. Tests run in order and build state sequentially.
