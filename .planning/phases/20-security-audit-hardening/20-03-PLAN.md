    ```html
    <head>
      <meta charset="utf-8" />
      <meta http-equiv="Content-Security-Policy" content="
        default-src 'self';
        script-src 'self';
        style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
        font-src 'self' https://fonts.gstatic.com;
        img-src 'self' data:;
        connect-src 'self' http://localhost:8082 http://localhost:8080 https://*.railway.app https://*.vercel.app;
        frame-src 'none';
        object-src 'none';
        base-uri 'self';
        form-action 'self';
      ">
      <title>Knowledge Hub</title>
      <base href="/" />
      <meta name="viewport" content="width=device-width, initial-scale=1" />
      <link rel="preconnect" href="https://fonts.googleapis.com" />
      <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
      <link rel="icon" type="image/svg+xml" href="favicon.svg" />
    </head>
    ```

    CSP rationale:
    - `default-src 'self'` — baseline, everything from same origin
    - `script-src 'self'` — no inline scripts (Angular uses bundled JS)
    - `style-src 'self' 'unsafe-inline'` — Angular/Tailwind injects styles inline; Google Fonts CSS loaded externally
    - `font-src 'self' https://fonts.gstatic.com` — Google Fonts
    - `img-src 'self' data:` — data: URIs for inline images (badges, etc.)
    - `connect-src 'self' http://localhost:8082 http://localhost:8080 https://*.railway.app https://*.vercel.app` — API connections (adjust for deployment URLs)
    - `frame-src 'none'; object-src 'none'` — no embedded content
    - `base-uri 'self'; form-action 'self'` — restrict base URL and form submissions

    **Step 2 — Add security headers via Spring Security (backend side)**

    In `SecurityConfig.java`, add a security headers filter that sets response headers for ALL requests. Add these headers to the HTTP response via the `HttpSecurity` config or a custom filter.

    Option A (Spring Security built-in, preferred): The `.headers()` configure in `SecurityConfig.java`:
    Add to the `filterChain` method:
    ```java
    http.headers(headers -> headers
        .httpStrictTransportSecurity(hsts -> hsts
            .includeSubDomains(true)
            .maxAgeInSeconds(31536000))
        .contentTypeOptions()
        .frameOptions(frame -> frame.deny())
        .xssProtection(xss -> xss
            .block(true))
    );
    ```

    However, note that `.xssProtection()` is deprecated in newer Spring Security (X-XSS-Protection is obsolete). The recommended approach is just HSTS and content-type options via the new DSL.

    For Spring Security 7.x (Boot 4.x uses Security 7.x):
    ```java
    http.headers(headers -> headers
        .httpStrictTransportSecurity(hsts -> hsts
            .includeSubDomains(true)
            .maxAgeInSeconds(31536000))
        .contentTypeOptions(Customizer.withDefaults())
        .frameOptions(frame -> frame.deny())
    );
    ```

    This sets:
    - `Strict-Transport-Security: max-age=31536000; includeSubDomains` (HSTS, 1 year)
    - `X-Content-Type-Options: nosniff` (content type sniffing prevention)
    - `X-Frame-Options: DENY` (clickjacking prevention)
