    **PART A: Fix DomSanitizer XSS vector in article-search.component.ts**

    Current code (line 176-184):
    ```typescript
    sanitizeHeadline(html: string): SafeHtml {
      if (html === '') return '';
      const cleaned = html
        .replace(/<(?!\/?mark(?=>|\s[^>]*>))[^>]*>/gi, '')
        .replace(/<mark\b[^>]*>/gi, '<mark>');
      const sanitized = this.sanitizer.sanitize(1 /* SecurityContext.HTML */, cleaned);
      return this.sanitizer.bypassSecurityTrustHtml(sanitized ?? cleaned);
    }
    ```

    The problem: `bypassSecurityTrustHtml` tells Angular "trust this HTML, don't sanitize it." While a regex pre-cleaning step exists, regex-based HTML sanitization is notoriously fragile (e.g., nested tags, encoded entities, zero-width characters).

    Fix approach:
    a. Install DOMPurify: `cd frontend && pnpm add dompurify && pnpm add -D @types/dompurify`
    b. Replace the sanitization logic:
    ```typescript
    import DOMPurify from 'dompurify';
    
    sanitizeHeadline(html: string): SafeHtml {
      if (html === '') return '';
      // DOMPurify: industry-standard HTML sanitization
      const clean = DOMPurify.sanitize(html, {
        ALLOWED_TAGS: ['mark'],
        ALLOWED_ATTR: [],
      });
      return this.sanitizer.bypassSecurityTrustHtml(clean);
    }
    ```
    c. Remove the `import { DomSanitizer, SafeHtml }` — keep `SafeHtml`, keep `DomSanitizer` since it's still used for `bypassSecurityTrustHtml` (but now after DOMPurify sanitization).

    **Note on risk acceptance:** We keep `bypassSecurityTrustHtml` because Angular has no way to render HTML with only certain allowed tags. But after DOMPurify sanitization, the input is guaranteed safe — only `<mark>` tags survive, no attributes allowed. The `bypassSecurityTrustHtml` is now a conscious decision with DOMPurify as the gatekeeper.

    **PART B: Manual frontend security audit**

    Read every frontend file systematically, organized by module, checking for security issues:

    **Core module (`core/`):**
    - `auth.service.ts`: Verify tokens are handled securely (HttpOnly cookies, no localStorage). Check that `tryRefreshToken()` doesn't expose refresh token in URL or body params insecurely.
    - `error.interceptor.ts`: Verify error responses don't leak sensitive data to console/logs. Check that 401 handling doesn't redirect to malicious URLs.
    - Guards (`auth.guard.ts`, `admin.guard.ts`, `agent.guard.ts`): Verify redirect URLs are hardcoded (not attacker-controlled). Verify no privilege escalation path.
    - `workspace.service.ts`: Verify workspace switching doesn't leak previous workspace data.
    - `translation.service.ts`: Verify no injection vectors in translation value rendering.

    **Feature modules (`features/`):**
    - **Auth** (`features/auth/`): Verify login/register forms handle errors safely. Check for any credential logging.
    - **Chat** (`features/chat/`): Verify AI response rendering — any `[innerHTML]` bindings that could contain malicious markdown? Check that markdown is rendered via `ngx-markdown` (which sanitizes output) and not raw innerHTML.
    - **KB articles** (`features/kb/`): All article content rendering — verify articles go through Angular's template binding (auto-sanitized by Angular). Check for any `bypassSecurityTrustHtml` or `[innerHTML]` that could receive user-controlled content.
    - **Agent dashboard** (`features/agent/`): Verify ticket content rendering is safe.
    - **Admin** (`features/admin/`): Verify user management UI doesn't expose sensitive fields. Check KCS draft rendering.
    - **Workspace management** (`features/workspace-switcher/`): Verify workspace names displayed safely.

    **Shared module (`shared/`):**
    - All pipes, directives, components: Check for any `bypassSecurityTrustHtml`, `bypassSecurityTrustUrl`, `bypassSecurityTrustStyle` usage.
    - Check custom pipes that transform user input.

    **App root:**
    - `app.config.ts`: Verify providers don't expose security-sensitive config.
    - `app.routes.ts`: Verify all routes use proper guards.

    **Security checklist for each file:**
    - [ ] Any `bypassSecurityTrust*` usage? → Must have sanitization before it
    - [ ] Any `[innerHTML]` bindings? → Check source is safe
    - [ ] Any `eval()` or `new Function()`? → Must be removed
    - [ ] Any `document.write()` or direct DOM manipulation? → Use Angular Renderer2
    - [ ] Any sensitive data in console.log? → Remove or change to warn/error
    - [ ] Any template expressions that could execute code? → Angular auto-sanitizes
    - [ ] Any unsafe URL handling? (javascript: URLs, open redirects)
    - [ ] Are all user-facing strings using TranslationService? (not an XSS vector, but i18n compliance)
