# New Account Creation

## Account Provisioning Process

When a new employee joins the company, the following accounts are automatically provisioned once HR initiates the onboarding workflow:

| Account Type | Automatically Created | Time to Provision |
|-------------|----------------------|-------------------|
| Network account (Active Directory) | Yes | 1 hour |
| Email (Microsoft 365) | Yes | 2 hours |
| VPN access | Yes | 2 hours |
| Intranet / Knowledge Base | Yes | 1 hour |
| File share access | By request | 1 business day |

## Role-Based Access Control (RBAC)

Access to systems is granted based on the employee's role and department:

- **Standard user** — email, intranet, VPN, basic file shares
- **Manager** — all standard + shared drive access, reporting tools
- **Department agent** — all standard + departmental applications, ticketing system
- **Administrator** — all systems with elevated privileges where required

## Submitting Additional Access Requests

If a new employee requires access beyond the standard provisioning, the hiring manager should:

1. Open a ticket through the IT Service Portal (https://helpdesk.company.com)
2. Select "Access Request" as the category
3. Specify the systems and access level required
4. Include the employee's name, department, and start date
5. Attach manager approval for elevated privileges

## Account Lifecycle

| Event | Action |
|-------|--------|
| Hire | Account created within 24 hours of start date |
| Transfer | Access reviewed and adjusted within 48 hours |
| Leave of absence | Account suspended after 30 days of inactivity |
| Termination | Account disabled within 4 hours of notification |

# Création de nouveaux comptes

## Processus de provisionnement des comptes

Lorsqu'un nouvel employé rejoint l'entreprise, les comptes suivants sont automatiquement provisionnés une fois que les RH ont initié le processus d'intégration :

| Type de compte | Créé automatiquement | Délai de provisionnement |
|----------------|----------------------|--------------------------|
| Compte réseau (Active Directory) | Oui | 1 heure |
| Courriel (Microsoft 365) | Oui | 2 heures |
| Accès VPN | Oui | 2 heures |
| Intranet / Base de connaissances | Oui | 1 heure |
| Accès aux partages de fichiers | Sur demande | 1 jour ouvrable |

## Contrôle d'accès basé sur les rôles (RBAC)

L'accès aux systèmes est accordé en fonction du rôle et du service de l'employé :

- **Utilisateur standard** — courriel, intranet, VPN, partages de fichiers de base
- **Gestionnaire** — tout ce qui est standard + accès aux lecteurs partagés, outils de rapport
- **Agent de service** — tout ce qui est standard + applications départementales, système de billetterie
- **Administrateur** — tous les systèmes avec privilèges élevés selon les besoins

## Soumission de demandes d'accès supplémentaires

Si un nouvel employé nécessite un accès au-delà du provisionnement standard, le responsable du recrutement doit :

1. Ouvrir un ticket via le Portail de services informatiques (https://helpdesk.company.com)
2. Sélectionner « Demande d'accès » comme catégorie
3. Spécifier les systèmes et le niveau d'accès requis
4. Inclure le nom, le service et la date d'entrée en fonction de l'employé
5. Joindre l'approbation du gestionnaire pour les privilèges élevés

## Cycle de vie des comptes

| Événement | Action |
|-----------|--------|
| Embauche | Compte créé dans les 24 heures suivant la date de début |
| Transfert | Accès révisé et ajusté dans les 48 heures |
| Congé | Compte suspendu après 30 jours d'inactivité |
| Départ | Compte désactivé dans les 4 heures suivant la notification |
